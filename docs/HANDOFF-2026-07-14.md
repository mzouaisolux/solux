## 6. Points de vigilance techniques (pièges)

- **5 maps events exhaustives** : tout nouvel `event_type` DOIT être ajouté dans `lib/events-shared.ts` (union + label map), `lib/events.ts` (DEFAULT_SEVERITY), `lib/notification-catalog.ts`, `lib/event-help.ts` (recipients ≥ 1, un test le vérifie). Sinon build KO.
- **Notifications = opt-in** : un event n'apparaît dans la cloche que si `event_routing(consumer='notification', role='*')` existe et est enabled. Le défaut dépend de migrations manuelles (m162/m168/m171). Table de lecture = `event_reads(user_id, event_id, last_read_at)`.
- **Capabilities** : grants dans la table **`role_permissions(role, permission_key)`** (PAS le catalogue `CAPABILITY_CATALOG`/`permissions` qui ne liste que les clés). `getCurrentUserRole()` → `{ userId, role, isSuperAdmin }`. `super_admin` = **booléen** sur `user_roles` (pas un rôle) ; seul super-admin prod = `mzouai@` (mdp inconnu de l'agent ; les tests promeuvent `testadmin` en LOCAL puis restaurent).
- **Pricing audit (#2)** : matching des lignes par `sr:{source_project_request_id}|{component}`, PAS par nom (attacher un modèle catalogue remet `client_product_name` à null). Flag `pricing.hide_catalogue_prices` (m142) : ON en prod → Sales ne voient pas prix/remise (défense) ; toggle local `e2e/audit/hide-prices-toggle.ts <0|1|read>`.
- **Gate pole drawing** : obligatoire seulement si `poleCostRmb > 0` (pas juste `pole_required`).
- **Wizard** : champ `light_mounting_height` est DANS le bloc `pole_required` (cocher d'abord). Enter sur étape non-finale ne soumet pas. `SubmitButton` a un garde d'hydratation (premier clic).
- **`updateProjectRequest`/`submitProjectRequest`** : guard optimiste `.eq("status","draft")`. Autres transitions (approve/requestMoreInfo/enterFactoryCost/setProjectPricing) : PAS de guard de concurrence (finding Moyen ouvert).
